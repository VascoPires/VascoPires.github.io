# Website Architecture Plan

## Positioning

Core story:
Vasco Pires is an engineer-researcher working at the intersection of composite materials, structural mechanics, aerospace-oriented design, computation, and technical communication.

The site should support two goals at the same time:
- academic credibility through research, publications, teaching, and talks
- selected external collaborations through clear evidence of engineering analysis, computational support, and technical writing work

The site should not read like a startup landing page or a long CV dump.
It should feel like a curated technical profile.

## Main Website Goals

1. Academic credibility
- Make current PhD, university role, publications, talks, and teaching easy to find.

2. Professional visibility
- Show applied engineering work, aerospace-related projects, and tools in a compact and credible way.

3. Selected consulting opportunities
- Signal availability for limited technical collaborations without over-commercializing the site.

4. Discoverability
- Help visitors quickly find projects, tools, writing, and resources from the homepage.

## Recommended Structure

### Home
Purpose:
- Answer who you are, what you work on, what you can help with, and where to go next.

Include:
- short positioning statement
- 3 to 4 technical focus areas
- selected work highlights
- compact collaboration block
- quick links to projects, publications, teaching, and contact

Show first:
- name, role, domain focus, current affiliation

Omit:
- long life story
- full lists of talks, publications, or every project

### About
Purpose:
- Provide fuller professional context and a restrained personal summary.

Include:
- short bio
- current role at Montanuniversitat Leoben
- background in mechanical engineering and aeronautical structures
- research interests
- compact skills/tools list
- short honors line

Show first:
- present role and technical focus

Omit:
- excessive narrative detail
- repeated CV content already available elsewhere

### Projects
Purpose:
- Curated overview page that summarizes what you do through selected projects and images.
- This should be a single strong page, not a click-heavy archive.

Include:
- 4 to 5 categories
- 1 to 3 flagship projects shown in more detail on the page itself
- grouped supporting projects as summary cards or compact rows
- 1 image, gif, figure, or diagram for selected items

Show first:
- a short intro explaining what kinds of work are included
- flagship projects that anchor your identity

Omit:
- too many nearly identical academic entries
- separate page for every project unless one later proves valuable

### Engineering / Technical Work
Purpose:
- Highlight practical engineering capability for external collaborators.

Include:
- engineering analysis
- structural design
- FEA/CAD/computation
- selected applied work from ESA, UAV, CAD reconstruction, 3D printing work

Show first:
- the most concrete applied examples

Omit:
- publication-style detail better kept in publications or talks

### Code / Tools
Purpose:
- Show computational support, automation, and engineering tooling.

Include:
- Python/Matlab tools
- engineering scripts
- tutorials or small interactive tools
- GitHub links

Show first:
- useful tools with practical value

Omit:
- placeholder repos
- notebook clutter without explanation

### Teaching / Resources
Purpose:
- Support academic visibility and show communication ability.

Include:
- course materials
- tutorials
- demos
- explanatory engineering resources

Show first:
- resources that are actually useful to students or engineers

Omit:
- fragmented notes that are not ready for public use

### Publications / Talks
Purpose:
- Provide academic proof and make outputs easy to browse.

Include:
- publications
- talks/posters
- citation profiles

Show first:
- selected recent outputs or filters

Omit:
- long project narratives

### News / Updates
Purpose:
- Small factual updates, not frequent blogging pressure.

Include:
- conference presentations
- paper acceptance
- project milestones
- award note

Show first:
- a few meaningful updates only

Omit:
- low-value status updates

### Contact / Work With Me
Purpose:
- Invite relevant collaborations in restrained language.

Include:
- concise collaboration statement
- email
- examples of help offered

Show first:
- what kinds of collaborations are a fit

Omit:
- sales language
- broad claims that sound like an agency

2. Focus areas

- computational tools and technical communication

3. Selected work

4. What I can help with
- technical analysis
- computational support
- scientific or technical writing
- selected research or engineering collaborations

5. Teaching and publications quick links

6. News or recent updates
- 2 to 4 items only

7. Contact

## Project Philosophy

Recommended approach:
- keep one main Projects page
- do not create a dedicated page for every project
- use a few flagship sections with text + media
- use grouped summary cards for the rest

This matches your preference and avoids overkill.

## Awards Placement

Best placement order:

1. Thesis project entry
- strongest context
- ties recognition to actual technical work

2. About page
- include one restrained honors sentence

3. CV page
- list formally under honors/awards

4. Homepage highlight
- only as a short line, not a hero banner

5. News/update post
- optional if you want a dated record

6. Dedicated honors page
- not needed now

Suggested wording:
- "The master's thesis work on curved composite joints received four awards and was presented at multiple conferences."

## What To Prioritize First

1. Rewrite homepage intro and structure
2. Replace current projects page with a curated overview page
3. Add 6 to 8 strong project summaries using the template file
4. Update About page with clearer positioning
5. Add restrained collaboration/contact section

## What Can Wait

- dedicated engineering page if the projects page already covers it well
- separate code/tools page if GitHub links are still sparse
- more frequent news posts
- detailed individual project pages

## What To Remove If The Site Gets Dense

- low-signal projects without a clear story or result
- old blog content unrelated to your technical identity
- repeated information across About, CV, and Projects
- too many homepage highlights
